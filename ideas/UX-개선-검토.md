## 1. 온보딩 플로우 개선

### 🔴 1-1. 온보딩 중간 이탈 복구 (Resume Onboarding)

**현재 문제:** 6단계 온보딩 중 이탈 시 처음부터 다시 시작해야 하는 구조. 오픈서베이 리포트에서 73.4%가 초기에 이탈한다고 했는데, 이탈 후 복귀 시 다시 처음부터 시작하면 재방문 의지가 꺾인다.

**개선안:**
- 각 단계 완료 시 로컬 저장 (localStorage/AsyncStorage)
- 재진입 시 "이어서 할까요?" 모달 표시
- 진행률 바를 상단에 표시해서 "거의 다 왔다" 인식 심기

```
┌─────────────────────────────────┐
│                                 │
│  ██████████░░░░  3/4 단계        │
│                                 │
│  지난번 이어서 할까요?            │
│                                 │
│  ┌─────────────────────────┐    │
│  │     이어서 하기            │    │
│  └─────────────────────────┘    │
│                                 │
│  처음부터 다시 할게요             │
│                                 │
└─────────────────────────────────┘
```

### 🔴 1-2. 스플래시에 소셜 로그인 명시

**현재 문제:** 스플래시에 "시작하기"와 "로그인"만 있고 소셜 로그인 옵션이 보이지 않음. 한국 사용자 대다수는 카카오/네이버 로그인을 기대한다.

**개선안:**
```
┌─────────────────────────────────┐
│                                 │
│         ◯                       │
│        먹루틴                    │
│                                 │
│   "건강한 식단, 실행까지 한 번에"   │
│                                 │
│   ┌─────────────────────────┐   │
│   │  🟡 카카오로 3초 시작       │   │
│   └─────────────────────────┘   │
│   ┌─────────────────────────┐   │
│   │  🟢 네이버로 시작           │   │
│   └─────────────────────────┘   │
│   ┌─────────────────────────┐   │
│   │  🍎 Apple로 시작           │   │
│   └─────────────────────────┘   │
│                                 │
│   이메일로 가입                   │
│                                 │
└─────────────────────────────────┘
```

### 🟡 1-3. 온보딩 갤러리에 자동 넘김(Auto-advance) 옵션

**현재 문제:** 갤러리 5장을 수동 스와이프해야 한다. 관심 없는 사용자에게는 5장이 길게 느껴질 수 있다.

**개선안:**
- 4초 후 자동 넘김 (수동 스와이프로 override 가능)
- "건너뛰기"를 우상단에 항상 노출 (현재도 있으나 유지)
- 하단에 "모두 보기 / 바로 시작하기" 2개 버튼 병행

### 🟡 1-4. 기본 정보 입력 시 슬라이더 UX

**현재 문제:** 0-4 기본 정보에서 나이/키/체중을 텍스트 입력으로 받고 있음. 모바일에서 숫자 키보드 전환이 번거롭다.

**개선안:**
- 스크롤 휠(picker) 또는 슬라이더 UI 적용
- 한국인 평균값을 기본값으로 세팅 (남성: 174cm/73kg, 여성: 161cm/57kg)
- 미세 조정은 +/- 버튼으로 가능하게

```
  키  ─────────●───── 170cm
               [-] [+]

  체중 ────────●────── 72kg
               [-] [+]
```

## 2. 주간 식단 캘린더 (Tab 1) 개선

### 🔴 2-1. 주말(토/일) 식단 누락

**현재 문제:** 주간 캘린더가 월~금만 표시. 주말에도 식사는 하는데 토/일 식단이 없다. 특히 3040 주부 타겟은 주말 식단이 핵심.

**개선안:**
- 기본: 7일 전체 표시 (좌우 스크롤)
- 주말은 색상 구분 (배경 톤 변경)
- 주말 메뉴는 "밀프렙 데이" 제안 (일요일 = 반찬 만들기 날)

```
│  ┌────┬────┬────┬────┬────┬────┬────┐ │
│  │ 월  │ 화  │ 수★ │ 목  │ 금  │ 토  │ 일  │ │
│  ├────┼────┼────┼────┼────┼────┼────┤ │
│  │점심│점심│점심│점심│점심│브런│🍳  │ │
│  │비빔│제육│된장│불고│잡채│치 │밀프│ │
│  │밥  │볶음│찌개│기  │밥  │    │렙  │ │
│  │520 │480 │380 │550 │490 │480 │DAY │ │
│  ├────┼────┼────┼────┼────┼────┼────┤ │
│  │저녁│저녁│저녁│저녁│저녁│저녁│저녁│ │
│  │닭갈│두부│고등│순두│김치│외식│집밥│ │
│  │비찜│조림│어조│부찌│찌개│    │한상│ │
│  │430 │350 │410 │360 │390 │FREE│520 │ │
│  └────┴────┴────┴────┴────┴────┴────┘ │
                    ← 스크롤 →
```

### 🔴 2-2. 실행 취소(Undo) 기능

**현재 문제:** 식단 변경(드래그 앤 드롭, 메뉴 변경) 후 되돌리기가 없다. 실수로 변경하면 원래 뭐였는지 기억 못 함.

**개선안:**
- 변경 직후 하단에 스낵바(toast) 표시: "점심 메뉴를 변경했어요 [되돌리기]"
- 5초간 표시 후 자동 사라짐
- 되돌리기 탭 시 즉시 원복

```
┌─────────────────────────────────┐
│                                 │
│  (기존 화면)                     │
│                                 │
│  ┌─────────────────────────┐    │
│  │ 점심을 된장찌개→순두부로    │    │
│  │ 변경했어요     [되돌리기]   │    │
│  └─────────────────────────┘    │
└─────────────────────────────────┘
```

### 🟡 2-3. 오늘 기준 자동 스크롤

**현재 문제:** 주간 캘린더 진입 시 항상 월요일부터 보여줌. 수요일에 앱을 열면 수요일이 바로 보여야 한다.

**개선안:**
- 오늘 날짜 자동 포커스 (★ 표시 + 살짝 확대)
- 좌우 스와이프로 다른 날 탐색

### 🟡 2-4. 식단 미리보기 이미지

**현재 문제:** 주간 캘린더에 음식 이름과 칼로리만 표시. 음식 사진이 없으면 시각적 매력이 떨어진다.

**개선안:**
- 각 셀에 작은 썸네일 (40x40px) 추가
- 이미지 없는 레시피는 음식 카테고리 아이콘 fallback

## 3. 장보기 탭 (Tab 2) 개선

### 🔴 3-1. 수동 아이템 추가 기능 부재

**현재 문제:** 장보기 리스트가 식단 기반 자동 생성만 지원. 우유, 휴지 같은 일상 필요 품목을 추가할 수 없다.

**개선안:**
- 리스트 하단에 "+ 직접 추가" 입력 필드
- "자주 사는 항목" 빠른 추가 칩
- 추가 항목은 "기타" 카테고리로 분류

```
│  ── 기타 (직접 추가) ────────── │
│  ☐ 우유           1개    ₩3,200│
│  ☐ 화장지         1팩    ₩9,900│
│                                 │
│  ┌─────────────────────────┐    │
│  │  + 항목 추가...            │    │
│  └─────────────────────────┘    │
│                                 │
│  자주 사는 항목:                  │
│  [우유] [계란] [식빵] [바나나]     │
```

### 🟡 3-2. 커머스별 가격 비교 한눈에 보기

**현재 문제:** 쿠팡/컬리/B마트 각각의 버튼만 있고, 어디가 더 싼지 한눈에 비교가 안 된다.

**개선안:**
- 총 예상 비용을 커머스별로 비교하는 섹션 추가
- 최저가 커머스 자동 추천 표시

```
│  ── 어디서 살까? ────────────── │
│  ┌──────────────────────────┐  │
│  │  🟠 쿠팡      ₩46,800      │  │
│  │  🟣 컬리      ₩51,200      │  │
│  │  🟢 B마트     ₩44,500 ⚡   │  │
│  │               ↑ 최저가      │  │
│  │                             │  │
│  │  💡 B마트가 ₩2,300 더 저렴   │  │
│  │     + 1시간 이내 배송        │  │
│  └──────────────────────────┘  │
```

### 🟡 3-3. 장보기 완료 체크 + 실구매 금액 입력

**현재 문제:** 장보기 리스트가 "체크리스트" 역할만 하고, 실제 구매 후 피드백이 없다. 예상 비용 vs 실제 비용 비교가 안 된다.

**개선안:**
- 장보기 완료 시 "실제 금액 입력" 유도
- 마이페이지의 "절약한 금액" 정확도 향상
- "이번 주 장보기 ₩45,200 완료! 예상보다 ₩3,000 절약" 피드백

## 4. 마이페이지 (Tab 3) 개선

### 🔴 4-1. 체중/건강 지표 추적 기능

**현재 문제:** 마이페이지에 성과(절약 금액, 나트륨 감소)는 있지만 체중 변화 그래프가 없다. 다이어트 목표 사용자에게 체중 추이 시각화는 핵심.

**개선안:**
- 주 1회 체중 입력 프롬프트 (푸시)
- 꺾은선 그래프로 체중 추이 시각화
- 목표 체중까지 남은 수치 표시

```
│  ── 체중 변화 ─────────────── │
│  ┌─────────────────────────┐  │
│  │  시작: 72.0kg → 현재: 70.2kg│
│  │  목표: 65.0kg (-5.0kg 남음) │
│  │                             │
│  │  72─┐                       │
│  │  71─│──┐                    │
│  │  70─│──│──●                 │
│  │     │──│──│──── 목표 65    │
│  │     1주 2주 3주              │
│  └─────────────────────────┘  │
```

### 🟡 4-2. 연속 기록 스트릭(Streak) 표시

**현재 문제:** 주간 달성률은 있지만 "연속 며칠 달성" 같은 게이미피케이션 요소가 약하다. 습관 형성에는 스트릭이 강력하다.

**개선안:**
- "🔥 12일 연속 식단 달성!" 배너
- 스트릭이 끊기려 할 때 푸시 알림
- 주간/월간 스트릭 달성 시 뱃지 부여

```
│  ┌──────────────────────────┐  │
│  │  🔥 12일 연속 식단 달성!     │  │
│  │                             │  │
│  │  월 화 수 목 금 토 일        │  │
│  │  🔥 🔥 🔥 🔥 🔥 🔥 🔥        │  │
│  │  🔥 🔥 🔥 🔥 🔥 ○ ○        │  │

│  │  최장 기록: 21일 (지난달)     │  │

### 🟡 4-3. 알림 설정 화면

**현재 문제:** 초안에서 푸시 알림 전략(식사 시간대별 추천, 주간 리포트 등)을 언급하지만, 마이페이지에 알림 설정 UI가 없다.

## 5. 콘텐츠 상세 (5단계 몰입 플로우) 개선

### 🔴 5-1. 현재 단계 진행 표시기

**현재 문제:** 5단계 세로 스크롤 구조인데 사용자가 현재 어디에 있는지 알 수 없다. 스크롤이 길어지면 "이게 끝이 없나?" 피로감 발생.

**개선안:**
- 우측에 플로팅 진행 표시기 (인디케이터)
- 탭하면 해당 단계로 스크롤 점프 가능

```
     본문                    인디케이터
┌──────────────────────┐    ●  ① 에피소드
│                      │    ○  ② 영양
│  ② 영양 스토리        │    ○  ③ 만드는 법
│  (현재 보고 있는 영역)  │    ○  ④ 완성
│                      │    ○  ⑤ 커머스
└──────────────────────┘
```

### 🔴 5-2. 레시피 타이머 완료 알림

**현재 문제:** STEP 4에 "5분 타이머 시작" 버튼이 있지만, 타이머 완료 시 어떤 피드백이 있는지 정의되지 않았다. 사용자가 다른 앱으로 이동한 경우 알림이 필수.

**개선안:**
- 타이머 완료 시 로컬 푸시 알림
- 진동 + 소리 알림
- "다음 STEP으로" 자동 유도
- 앱 내에서도 타이머 진행 상태 플로팅 표시

```
┌─────────────────────────────────┐
│  ┌──────────────────────────┐   │
│  │  ⏱ 03:42 남음   [일시정지]  │   │
│  └──────────────────────────┘   │
│                                 │
│  (다른 STEP 보는 중에도          │
│   상단에 타이머 플로팅)           │
│                                 │
└─────────────────────────────────┘
```

### 🟡 5-3. 레시피 난이도별 필터링

**현재 문제:** 메뉴 변경 시 칼로리 유사 메뉴만 추천. 요리 초보자는 난이도 ★☆☆ 위주로 보고 싶을 수 있다.

**개선안:**
- 메뉴 변경 시트에 난이도 필터 추가
- "초보 모드" 토글: 난이도 ★★★ 메뉴 숨기기
- 온보딩 식사 스타일에서 요리 수준을 묻고 반영

### 🟡 5-4. 레시피 완료 체크 + 사진 인증

**현재 문제:** "나도 만들었다!" 버튼이 있지만 SNS 공유만 연결. 앱 내에서 완료 기록이 남지 않는다.

**개선안:**
- "만들었어요" 체크 → 식단 달성률에 반영
- 선택적 사진 촬영 → 레시피 리뷰에 자동 등록
- 별점 + 한 줄 후기 입력 유도

## 6. 냉장고 파먹기 개선

### 🟡 6-1. 사진 인식 재료 입력

**현재 문제:** 재료를 수동 검색/선택해야 함. 냉장고 사진 찍으면 AI가 재료를 자동 인식하면 훨씬 빠르다.

**개선안:**
- "📷 냉장고 사진 찍기" 버튼 추가 (Phase 2)
- AI 인식 결과 → 사용자 확인/수정 → 레시피 매칭
- 초안에서 이미 AI 사진 인식을 언급했으나 냉장고 파먹기와 연결하지 않았음

### 🟡 6-2. 유통기한 임박 재료 우선 추천

**현재 문제:** 냉장고 재료 입력 시 유통기한 정보가 없음. "곧 상할 재료"를 먼저 쓰는 레시피를 우선 추천하면 음식 낭비 줄임.

**개선안:**
- 재료 등록 시 선택적 유통기한 입력
- "⚠️ 3일 내 소진 추천" 필터
- 마이페이지에 "음식물 쓰레기 절약량" 지표 추가

## 7. 빈 상태(Empty State) 및 에러 처리

### 🔴 7-1. 빈 상태 화면 설계 누락

**현재 문제:** 신규 사용자가 온보딩 후 식단 탭을 열었을 때 식단이 아직 없는 상태, 레시피 컬렉션이 비어 있는 상태 등에 대한 화면이 없다.

**개선안:**

```
── 식단 탭 (빈 상태) ──────────────

┌─────────────────────────────────┐
│                                 │
│        🍽️                       │

│                                 │
│   건강 목표에 맞는 이번 주 식단을  │
│   자동으로 만들어드릴까요?         │
│                                 │
│   ┌─────────────────────────┐   │
│   │   이번 주 식단 만들기 →     │   │
│   └─────────────────────────┘   │
│                                 │
│   또는 레시피부터 저장하기 →      │
│                                 │
└─────────────────────────────────┘

── 레시피 컬렉션 (빈 상태) ─────────

┌─────────────────────────────────┐
│                                 │
│        📒                       │
│   저장한 레시피가 없어요          │
│                                 │

│   레시피를 저장해보세요            │
│                                 │
│   ┌─────────────────────────┐   │
│   │   익스텐션 설치 가이드 →    │   │

│   인기 레시피 둘러보기 →          │

```

### 🔴 7-2. 네트워크 에러 / 로딩 실패 화면

**현재 문제:** 영양정보 API 호출 실패, 커머스 링크 연결 실패 등의 에러 상태가 정의되지 않았다.

**개선안:**
- 공통 에러 화면 컴포넌트 정의
- 재시도 버튼 + 오프라인 캐시 데이터 표시
- 커머스 링크 실패 시 대체 검색 링크 제공

```

│                                 │

│                                 │

## 8. 접근성(Accessibility) 개선

### 🟡 8-1. 별점/난이도 접근성

**현재 문제:** 난이도 ★★☆, 별점 ⭐ 4.7 등이 시각적 표현에만 의존. 스크린 리더 사용자가 인식 불가.

**개선안:**
- `aria-label="난이도 5점 만점 중 2점"` 추가
- 색상 + 아이콘 + 텍스트 병행 표기 (예: "★★☆ 보통")

### 🟡 8-2. 나트륨 경고의 색상 의존 탈피

**현재 문제:** ⚠️ 나트륨 경고가 색상(노랑/빨강)에 의존할 가능성. 색약/색맹 사용자는 구분 불가.

**개선안:**
- 경고 수준을 텍스트로 명시: "주의" / "위험"
- 아이콘 형태도 차별화: ⚠️ (주의) vs 🚫 (위험)
- 진행바에 패턴(줄무늬 등) 추가

### 🟢 8-3. 글자 크기 조절

**개선안:**
- 마이페이지 설정에 글자 크기 조절 옵션 (소/중/대)
- 특히 3040 이상 사용자 + 요리 중 레시피 확인 시 큰 글자 필요

## 9. 마이크로 인터랙션 및 피드백

### 🟡 9-1. 식단에 메뉴 추가/변경 시 피드백

**현재 문제:** 냉장고 파먹기에서 "식단에 추가" 버튼이 있지만, 추가 후 피드백이 정의되지 않았다.

**개선안:**
- 추가 성공 시 토스트: "수요일 점심에 추가했어요! [보기]"
- 메뉴 변경 시 칼로리 변화 애니메이션: "380kcal → 360kcal (-20)"
- 드래그 앤 드롭 시 햅틱 피드백 (모바일)

### 🟡 9-2. 장보기 아이템 체크 시 만족감

**현재 문제:** 체크박스를 탭하면 그냥 체크될 뿐, 추가적인 피드백이 없다.

**개선안:**
- 체크 시 취소선 + 살짝 페이드 처리
- 전체 완료 시 축하 애니메이션: "이번 주 장보기 완료!"
- 진행 표시: "12/16 구매 완료"

### 🟢 9-3. 스크롤 방향 힌트

**현재 문제:** 5단계 몰입 플로우에서 "↓ 이 음식의 비밀", "↓ 직접 만들어보기" 등 텍스트 힌트는 있지만 시각적으로 약하다.

**개선안:**
- 바운스 애니메이션으로 아래 방향 유도
- 다음 섹션 살짝 보이기 (peek) 처리
- 스크롤 감지 후 힌트 자동 숨김

## 10. 개인화 및 학습 기능

### 🟡 10-1. 식단 선호도 학습 ("좋아요/싫어요")

**현재 문제:** 식단 추천은 칼로리/영양소 기반이지만, 사용자 취향(매운 거 싫어요, 생선 좋아요 등)을 반영하는 피드백 루프가 없다.

**개선안:**
- 각 메뉴 카드에 👍/👎 (또는 ♡) 버튼
- 싫어요 선택 시 "어떤 점이 아쉬웠나요?" 간단 설문
  - 재료가 싫어요 / 조리가 복잡해요 / 맛이 안 맞을 것 같아요
- 피드백 데이터로 추후 추천 알고리즘 개선

### 🟡 10-2. "이미 만들어 본" 레시피 표시

**현재 문제:** 레시피 컬렉션에 저장한 레시피/등록한 레시피 구분은 있지만, "만들어 본 적 있는지" 여부가 없다.

**개선안:**
- 레시피 상태: 저장됨 → 시도함 → 자주 만듦
- 식단 추천 시 "아직 안 만들어 본 레시피"를 우선 추천하거나 필터
- "이번 달 새로운 레시피 7개 도전!" 같은 동기부여 요소

### 🟢 10-3. 식사 스타일 변경의 유연성

**현재 문제:** 온보딩에서 건강 목표/식사 스타일을 설정하지만, 상황이 바뀔 수 있다 (출장, 명절, 다이어트 변경 등).

**개선안:**
- "이번 주만 모드 변경" 기능
  - 출장 모드: 경로 B(배달/외식) 위주 추천
  - 명절 모드: 칼로리 제한 완화
  - 집중 모드: 칼로리 제한 강화
- 마이페이지에서 상시 목표 수정 가능 (현재도 있으나 "모드 변경" 개념 추가)
