# 먹루틴 (MealRoutine) — 서비스 기획 초안

> **출처:** overseas-service 크론잡 수집 (2026-02-21)
> **우선순위:** 1순위 — 커머스 연동 수익화 명확, 기술 장벽 낮음
> **최종 수정:** 2026-02-21 — 스토리텔링 + 출처 기반 신뢰구조 + 한국형 벤치마크 반영

---

## 한 줄 소개

다이어트·운동·건강 목표에 맞춰 직접 요리하거나 바로 주문하고, 모든 경로에서 커머스 수익이 발생하는 식생활 플랫폼

---

## 0. 왜 지금, 이 서비스가 필요한가 — 숫자가 말하는 당위성

### 장면 1: "오늘 점심 뭐 먹지?"

> 서울 강남의 5년차 직장인 A씨(31)는 매일 점심시간이 스트레스다. 평균 점심값은 **9,500원**, 삼성동 기준 **15,000원**. 한 달이면 **20만원**이 훌쩍 넘는다. 도시락을 싸면 월 10만원을 아낄 수 있다는 걸 알지만, "이번 주 뭘 해먹지?" → "재료는 뭘 사지?" → "장은 어디서 보지?"를 매번 따로 고민하다 결국 배달 앱을 연다.
>
> — *NHN페이코 2025 상반기 모바일 식권 900만 건 분석, [스포츠경향 2025.07](https://sports.khan.co.kr/article/202507011042003)*

**직장인 72%가 점심값을 부담스럽게 느끼고, 78%가 주 2회 이상 도시락을 시도하지만** 식단 계획의 번거로움 때문에 지속하지 못한다.

### 장면 2: "배달시켜 먹었는데, 건강이..."

> 1인가구 B씨(28)는 주 4회 배달 음식을 시킨다. 편하지만 찝찝하다. 1인가구의 배달·포장 음식 나트륨 섭취량은 **2019년 146mg → 2022년 301mg으로 2배 이상 급증**. 한국인 평균 나트륨 섭취량은 **3,136mg/일**로 WHO 권고(2,000mg)의 **1.5배**를 초과한다. 의학 전문가들은 이를 "영양의 폭력"이라 부른다.
>
> — *[서울경제 2024](https://m.sedaily.com/amparticle/13897433), [질병관리청 국민건강영양조사 2023](https://www.kdca.go.kr/board/board.es?mid=a20501010000&bid=0015&list_no=726622&act=view)*

**2030 세대에서 당뇨·고혈압 등 "노인성 만성질환"이 급증**하고 있으며, 초가공식품 섭취량은 역대 최고치를 기록했다.

### 장면 3: "식단 관리 앱 깔았는데, 결국 삭제..."

> 다이어트를 결심한 C씨(34)는 밀리그램·눔을 깔았다. 기록은 잘 된다. 그런데 **"그래서 오늘 뭘 먹으라는 건데?"**에 대한 답이 없다. 레시피를 찾으려면 만개의레시피, 장을 보려면 쿠팡, 배달을 시키려면 배민... 앱 3개를 넘나들다 지쳐서 기록도 그만둔다.
>
> — *밀리그램 120만 다운로드, 유저 75%가 2.9kg 감량 ([한국경제 2023](https://plus.hankyung.com/apps/newsinside.view?aid=202303084308i)). 그러나 식단 앱 3개월 유지율은 업계 평균 20% 미만.*

**"기록"에서 "실행"까지의 연결이 끊겨 있는 것이 핵심 문제.**

### 장면 4: "반찬 뭐 하지?"

> 두 아이 엄마 D씨(38)는 매주 일요일이 고민이다. "이번 주 반찬 뭐 하지?" 만개의레시피에서 레시피를 찾고, 메모장에 재료를 옮겨 적고, 마켓컬리에서 하나씩 검색한다. 이 과정만 1시간. 한식은 한 끼에 반찬이 3~4개니 매주 반복되는 노동이다.
>
> — *한국 가정의 한 끼 구성: 밥 + 국 + 반찬 2~3개. 주간 단위 반찬 계획은 한식 고유의 니즈이나, 이를 지원하는 서비스가 전무.*

---

### 이 4가지 장면의 공통점

| 문제 | 현재 해결책 | 한계 |
|------|-----------|------|
| 뭘 해먹지? | 만개의레시피 | 식단 계획·장보기 연결 없음 |
| 건강하게 먹고 싶다 | 눔·밀리그램 | "기록"만 있고 "실행" 없음 |
| 장보기 귀찮다 | 쿠팡·컬리 | 레시피·건강 목표 연결 없음 |
| 시간 없어서 시킨다 | 배민·쿠팡이츠 | 건강 필터 없음 |

**먹루틴은 이 4가지를 하나로 잇는다.**

## 핵심 구조

사용자의 진입점은 **건강 목표(다이어트, 운동, 건강 관리)**이고, 거기서 두 갈래 경로로 분기하며 **양쪽 모두 어필리에이트 수익**이 발생하는 구조:

```
                    ┌─────────────────────┐
                    │  사용자 건강 목표      │
                    │  다이어트 / 운동 / 건강 │
                    └──────────┬──────────┘
                               │
                    ┌──────────▼──────────┐
                    │  맞춤 식단 추천        │
                    │  (칼로리·영양소 기반)   │
                    │  📊 출처: 식약처 DB    │
                    └──────────┬──────────┘
                               │
              ┌────────────────┴────────────────┐
              ▼                                  ▼
    ┌─────────────────┐                ┌─────────────────┐
    │  경로 A: 직접 요리  │                │  경로 B: 바로 주문  │
    │  "오늘은 해먹자"    │                │  "오늘은 시키자"    │
    └────────┬────────┘                └────────┬────────┘
             │                                   │
    ┌────────▼────────┐                ┌────────▼────────┐
    │ 레시피 제공        │                │ 음식 추천          │
    │ (냉장고 파먹기 포함) │                │ (건강 목표 기반)    │
    │ 📎 출처 링크 표시   │                │ 📊 영양정보 표시    │
    └────────┬────────┘                └────────┬────────┘
             │                                   │
    ┌────────▼────────┐                ┌────────▼────────┐
    │ 식재료 장보기       │                │ 음식 주문          │
    │ 쿠팡 / 컬리 / B마트 │                │ 배민 / 쿠팡이츠     │
    └────────┬────────┘                └────────┬────────┘
             │                                   │
    ┌────────▼────────┐                ┌────────▼────────┐
    │ 💰 식재료 어필리에이트│                │ 💰 음식 어필리에이트 │
    │ (구매당 수수료)     │                │ (주문당 수수료)     │
    └─────────────────┘                └─────────────────┘
```

### 왜 이 구조인가?

현실에서 사용자는:
- 시간 있을 때 → 직접 요리 (월 10만원 절약 가능 — 한국소비자원)
- 바쁠 때 → 배달 주문 (평균 9,800원/끼 — NHN페이코)

**두 상황 모두에서 "건강한 선택"을 도와주고, 양쪽 모두에서 수익이 발생**하는 구조가 핵심.

### 핵심 차별점

> 위 서비스들은 모두 **"기록"** 또는 **"콘텐츠"**에서 멈춘다. 먹루틴은 **"기록이 아니라 실행"** — 건강 목표에 맞는 레시피를 추천하고, 재료를 장바구니에 담아주거나, 건강한 배달 메뉴를 바로 주문할 수 있게 연결한다.

## 한국에 없는 이유

각 영역의 강자가 따로 존재하지만, **사일로(silo)로 분리**되어 있음:

```
현재 한국 시장의 사일로 구조:

[만개의레시피]──── 레시피만 ────────────────────── 끝.
[밀리그램/눔] ──── 기록만 ─────────────────────── 끝.
[쿠팡/컬리] ───── 구매만 ─────────────────────── 끝.
[배민/쿠팡이츠] ── 배달만 ─────────────────────── 끝.

먹루틴이 잇는 플로우:

건강 목표 → 식단 추천 → 레시피 or 메뉴 → 장보기 or 주문 → 💰
```

**건강 목표 → 실행(요리/주문) → 커머스 수익**을 하나로 잇는 서비스가 없음.

## 데이터 신뢰 구조 — "출처 있는 건강 정보"

먹루틴이 다루는 레시피·영양·건강 정보는 반드시 **검증된 출처**에서 가져오고, 사용자에게 출처를 투명하게 표시하는 구조:

### 데이터 소스별 출처 체계

```
┌───────────────────────────────────────────────────────────┐
│  📊 영양 정보 — 공공 데이터 (출처 100% 명시)                  │
│  ├── 식약처 식품영양성분DB (data.go.kr/data/15127578)         │
│  │   → 26개 영양소, 무료 API, "출처: 식품의약품안전처" 자동 표기 │
│  ├── 전국통합식품영양성분 표준데이터 (data.go.kr/data/15100064) │
│  └── 식약처 맞춤 영양 진단 모델 (소스코드 공개)                 │
│                                                             │
│  📎 레시피 — 3계층 출처 관리                                  │
│  ├── Layer 1: 공공데이터포털 레시피 DB (식약처/농림수산식품)     │
│  │   → 출처: "공공데이터포털 / 식품의약품안전처" 자동 표기       │
│  ├── Layer 2: 사용자 저장 (크롬 익스텐션)                      │
│  │   → 원본 URL 자동 저장 → "출처: 만개의레시피 (원본 보기)" 링크│
│  │   → 재료·순서만 구조화, 원본 사진·서술 미복제               │
│  └── Layer 3: 사용자 직접 등록 (UGC)                          │
│      → 출처: "사용자 직접 등록" 표기                           │
│                                                             │
│  🏷️ 건강 콘텐츠 — 스토리 소재의 출처                          │
│  ├── 질병관리청 국민건강영양조사 (연 1회 공식 발표)              │
│  ├── 식품의약품안전처 나트륨·당류 섭취 실태 보고서               │
│  ├── 통계청 1인가구 통계 (대한민국 정책브리핑)                  │
│  └── 뉴스 기사 (서울경제, 헤럴드경제, 스포츠경향 등)            │
│      → 모든 수치에 출처 표기, 링크 제공                        │
└───────────────────────────────────────────────────────────┘
```

│                                           │

│  조리 순서: 1. 두부를 깍둑 썰기 ...         │
│                                           │
│  ─── 영양 정보 ──────────────────────────  │
│  열량 380kcal | 단백질 18g | 나트륨 890mg  │
│  ⚠️ 나트륨 WHO 1일 권고량의 45%           │
│                                           │
│  ─── 출처 ───────────────────────────────  │
│  📎 레시피 원본: 만개의레시피 (원본 보기 →)  │
│  📊 영양정보: 식품의약품안전처 식품영양성분DB │
│  📋 공공누리 제1유형                        │

### 스토리텔링 플로우 (온보딩)

```
Step 1: 공감 (뉴스/통계 기반)
    "한국인 평균 나트륨 섭취, WHO 권고의 1.5배입니다"
    — 출처: 질병관리청 국민건강영양조사 2023

Step 2: 문제 인식
    "배달 음식 나트륨, 3년 만에 2배 증가했습니다"
    — 출처: 서울경제 2024

Step 3: 해결 제시
    "먹루틴이 당신의 건강 목표에 맞는 식단을 추천합니다"
    → 직접 요리할 시간이 있다면 → 레시피 + 장보기
    → 바쁘다면 → 건강한 배달 메뉴 추천

Step 4: 신뢰
    "모든 영양 정보는 식품의약품안전처 공공 데이터 기반입니다"
    → 출처 표시 + 공공누리 마크
```

## 한국화 포인트

- **한식 반찬 세트 단위 플래닝:** 한 끼 = 밥 + 국 + 반찬 2~3개 → "이번 주 반찬 세트" 단위로 식단 계획
- **냉장고 파먹기 모드:** 남은 재료 입력 → 건강 목표에 맞는 레시피 추천 → 부족한 재료만 장보기 연동
- **가구 분량 자동 계산:** 1인 / 2인 / 가족 분량 자동 조절
- **듀얼 커머스 연동:** 식재료(쿠팡/컬리/B마트) + 음식 주문(배민/쿠팡이츠) 양방향
- **나트륨·당류 경고:** 식약처 기준 초과 시 시각적 경고 표시 (출처 명시)

- **사용자가 요리하든 주문하든 수익 발생** → 단일 경로 의존 리스크 없음

- 스토리텔링 콘텐츠(뉴스·통계 기반)가 **SEO 트래픽 + 신규 유입** 채널 역할

## 경쟁 우위

1. **듀얼 어필리에이트:** 요리/주문 양쪽 모두에서 수익 → 사용자 행동에 관계없이 수익화
2. **"기록 → 실행" 전환:** 밀리그램(120만)·눔이 못 하는 "그래서 뭘 먹지?" 해결
3. **출처 기반 신뢰:** 식약처 공공 데이터 + 뉴스 출처 → 건강 정보 신뢰도 확보
4. **스토리텔링 SEO:** 뉴스·통계 기반 콘텐츠로 검색 유입 → 서비스 전환
5. **냉장고 파먹기 → 장보기:** 절약 체감 → 재방문율 상승
6. **선점 효과:** 건강·레시피·커머스를 잇는 포지션 → 밀키트 구독 박스 확장

## MVP 범위

> **목표:** 노코드 또는 소규모 팀으로 4~6주 내 검증 가능

### Phase 1: 경로 A (직접 요리) 우선 검증

1. **크롬 익스텐션** — 웹 레시피 원클릭 저장 (원본 URL 출처 자동 보존)
2. **재료 자동 추출** — 저장된 레시피에서 재료 자동 파싱
3. **주간 식단 캘린더** — 끌어다 놓기 방식 + 칼로리/영양소 합산 표시 (식약처 DB 기반)
4. **냉장고 파먹기** — 보유 재료 입력 → 레시피 매칭
5. **장보기 리스트** — 재료 취합 → 쿠팡 검색 링크 자동 생성 (API 없이도 가능)
6. **스토리텔링 랜딩** — 뉴스·통계 기반 당위성 콘텐츠 (SEO용)

### Phase 2: 경로 B (바로 주문) 추가

7. **건강 목표 설정** — 다이어트/벌크업/유지 목표 + 칼로리 설정
8. **음식 추천** — 목표에 맞는 외식/배달 메뉴 추천 (영양 정보 출처 표시)
9. **배달 앱 연동** — 배민/쿠팡이츠 검색 링크 자동 생성
