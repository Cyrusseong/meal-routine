# 먹루틴 (MealRoutine) — 서비스 기획 초안

> **출처:** overseas-service 크론잡 수집 (2026-02-21)
> **우선순위:** 1순위 — 커머스 연동 수익화 명확, 기술 장벽 낮음
> **최종 수정:** 2026-02-21 — 핵심 구조 재정의

---

## 한 줄 소개

다이어트·운동·건강 목표에 맞춰 직접 요리하거나 바로 주문하고, 모든 경로에서 커머스 수익이 발생하는 식생활 플랫폼

---

## 핵심 구조

사용자의 진입점은 **건강 목표(다이어트, 운동, 건강 관리)**이고, 거기서 두 갈래 경로로 분기하며 **양쪽 모두 어필리에이트 수익**이 발생하는 구조:

```
                    ┌─────────────────────┐
                    │  사용자 건강 목표      │
                    │  다이어트 / 운동 / 건강 │
                    └──────────┬──────────┘
                               │
                    ┌──────────▼──────────┐
                    │  맞춤 식단 추천        │
                    │  (칼로리·영양소 기반)   │
                    └──────────┬──────────┘
                               │
              ┌────────────────┴────────────────┐
              ▼                                  ▼
    ┌─────────────────┐                ┌─────────────────┐
    │  경로 A: 직접 요리  │                │  경로 B: 바로 주문  │
    │  "오늘은 해먹자"    │                │  "오늘은 시키자"    │
    └────────┬────────┘                └────────┬────────┘
             │                                   │
    ┌────────▼────────┐                ┌────────▼────────┐
    │ 레시피 제공        │                │ 음식 추천          │
    │ (냉장고 파먹기 포함) │                │ (건강 목표 기반)    │
    └────────┬────────┘                └────────┬────────┘
             │                                   │
    ┌────────▼────────┐                ┌────────▼────────┐
    │ 식재료 장보기       │                │ 음식 주문          │
    │ 쿠팡 / 컬리 / B마트 │                │ 배민 / 쿠팡이츠     │
    └────────┬────────┘                └────────┬────────┘
             │                                   │
    ┌────────▼────────┐                ┌────────▼────────┐
    │ 💰 식재료 어필리에이트│                │ 💰 음식 어필리에이트 │
    │ (구매당 수수료)     │                │ (주문당 수수료)     │
    └─────────────────┘                └─────────────────┘
```

### 왜 이 구조인가?

기존 초안은 "레시피 → 장보기"의 단일 경로였음. 하지만 현실에서 사용자는:
- 시간 있을 때 → 직접 요리
- 바쁠 때 → 배달 주문

**두 상황 모두에서 "건강한 선택"을 도와주고, 양쪽 모두에서 수익이 발생**하는 구조가 핵심.

---

## 벤치마크

| 항목 | 내용 |
|---|---|
| **원본 서비스** | Paprika (미국) — 레시피 + 식단 + 장보기 통합 |
| **추가 벤치마크** | MyFitnessPal — 건강 목표 → 식단 추천 진입점 |
| **추가 벤치마크** | Noom — 다이어트 코칭 → 식생활 개선 플로우 |
| **차별점** | 위 서비스들은 "기록"에 집중. 먹루틴은 **기록이 아니라 "실행"(요리 or 주문)까지 연결** |

---

## 한국에 없는 이유

- 만개의레시피·해먹남녀 = 레시피만 (건강 목표 연결 없음, 커머스 연동 없음)
- 눔·밀리그램 = 기록만 (실행 연결 없음, "그래서 뭘 먹지?"에 답 없음)
- 배민·쿠팡이츠 = 주문만 (건강 목표 연결 없음)

**건강 목표 → 실행(요리/주문) → 커머스 수익**을 하나로 잇는 서비스가 없음.

---

## 한국화 포인트

- **한식 반찬 세트 단위 플래닝:** 한 끼 = 밥 + 국 + 반찬 2~3개 → "이번 주 반찬 세트" 단위로 식단 계획
- **냉장고 파먹기 모드:** 남은 재료 입력 → 건강 목표에 맞는 레시피 추천 → 부족한 재료만 장보기 연동
- **가구 분량 자동 계산:** 1인 / 2인 / 가족 분량 자동 조절
- **듀얼 커머스 연동:** 식재료(쿠팡/컬리/B마트) + 음식 주문(배민/쿠팡이츠) 양방향

## 타겟 유저

| 세그먼트 | 주 경로 | 핵심 니즈 |
|---|---|---|
| **2030 직장인** | 경로 B(주문) 위주, 주말 경로 A(요리) | "바쁜데 건강하게 먹고 싶다" |
| **3040 주부** | 경로 A(요리) 위주 | "이번 주 반찬 뭐 하지? + 장보기 귀찮다" |
| **헬스/다이어트** | 경로 A+B 혼합 | "목표 칼로리에 맞는 식단이 필요하다" |

---

## 수익 모델

| 구분 | 경로 | 내용 |
|---|---|---|
| **식재료 어필리에이트** | 경로 A (직접 요리) | 쿠팡 파트너스 / 컬리 제휴 — 장보기 클릭·구매당 수수료 |
| **음식 주문 어필리에이트** | 경로 B (바로 주문) | 배민 / 쿠팡이츠 제휴 — 주문 연결당 수수료 |
| **프리미엄 구독** | 공통 | 월 3,900원 — 무제한 레시피 + 고급 영양 분석 + AI 식단 코칭 |
| **확장 모델** | 경로 A 확장 | 식재료 큐레이션 구독 박스 (밀키트 버전) |

### 수익 구조 특징

- **사용자가 요리하든 주문하든 수익 발생** → 단일 경로 의존 리스크 없음
- 경로 A 사용자는 식재료 어필리에이트, 경로 B 사용자는 음식 주문 어필리에이트
- 건강 목표라는 진입점이 **자연스러운 구매 전환율** 확보 (목적이 명확한 사용자)

---

## 경쟁 우위

1. **듀얼 어필리에이트:** 요리/주문 양쪽 모두에서 수익 → 사용자 행동에 관계없이 수익화
2. **건강 목표 → 실행 연결:** 기록 앱(눔)이나 레시피 앱(만개의레시피)이 못 하는 "실행"까지 연결
3. **냉장고 파먹기 → 장보기:** 있는 재료 활용 + 부족분만 주문 → 절약 체감 → 재방문율 상승
4. **선점 효과:** 레시피·건강·커머스를 잇는 포지션 선점 → 이후 밀키트 구독 박스로 확장

## MVP 범위

> **목표:** 노코드 또는 소규모 팀으로 4~6주 내 검증 가능

### Phase 1: 경로 A (직접 요리) 우선 검증

1. **크롬 익스텐션** — 웹 레시피 원클릭 저장
2. **재료 자동 추출** — 저장된 레시피에서 재료 자동 파싱
3. **주간 식단 캘린더** — 끌어다 놓기 방식 + 칼로리/영양소 합산 표시
4. **냉장고 파먹기** — 보유 재료 입력 → 레시피 매칭
5. **장보기 리스트** — 재료 취합 → 쿠팡 검색 링크 자동 생성 (API 없이도 가능)

### Phase 2: 경로 B (바로 주문) 추가

6. **건강 목표 설정** — 다이어트/벌크업/유지 목표 + 칼로리 설정
7. **음식 추천** — 목표에 맞는 외식/배달 메뉴 추천
8. **배달 앱 연동** — 배민/쿠팡이츠 검색 링크 자동 생성
