### 1.2 한국 시장 트렌드

| 트렌드 | 상세 |
|--------|------|
| **온라인 식품 시장** | 2025년 약 52조 원 규모 |
| **신선 식재료 직접 구매 증가** | 곡물(+17.9%), 채소(+20.9%), 과일(+19.8%) — 직접 요리 경향 강화 |
| **간편식(HMR) 온라인 구매 감소** | -8.4% — 소비자가 직접 요리하는 추세로 전환 |
| **1인 가구 증가** | 한 그릇 메뉴(덮밥 +8.2%, 샐러드 +22.2%) 성장 |

| **식료품 배달 연동** | 배달 서비스 연동 채택률 48% 달성 |

**시장 기회:** 한국에서 "레시피 검색 → 식단 계획 → 장보기 연동"의 통합 플로우를 제공하는 서비스가 부재. 직접 요리 트렌드 강화와 온라인 식품 시장 성장이 맞물려 타이밍이 유리함.

## 3. 커머스 API 연동 가능성

### 3.1 쿠팡 파트너스

| 항목 | 상세 |
|------|------|
| **공식 API** | ✅ 존재 — developers.coupangcorp.com |
| **가입 조건** | 2단계: 계정 생성 → 첫 매출 발생 시 최종 승인 |
| **API 활성화 조건** | 판매물품 총 금액 **15만원 이상** 달성 후 API 키 발급 가능 |
| **지원 언어** | Java, Python, PHP, C#, **Node.js** |
| **키 형태** | Access Key + Secret Key (HMAC 인증) |
| **MVP 우회 전략** | API 없이도 쿠팡 검색 URL 자동 생성으로 장보기 연동 가능 |
| **실현 가능성** | ⭐⭐⭐⭐ — MVP는 검색 링크로 시작, 매출 발생 후 API 전환 |

### 3.2 마켓컬리

| 항목 | 상세 |
|------|------|
| **공식 파트너 API** | ❌ 공개된 외부 개발자용 API 없음 |
| **파트너 오피스** | 3p-partner.kurly.com (입점 파트너 전용) |
| **기술 스택** | Vue.js (프론트), PHP/Laravel + Java/Spring Boot (백엔드) |
| **MVP 우회 전략** | 마켓컬리 검색 URL 자동 생성 (https://www.kurly.com/search?sword=재료명) |
| **실현 가능성** | ⭐⭐⭐ — 공식 API 부재로 검색 링크 방식이 현실적. 향후 사업 제휴 필요 |

### 3.3 배민 B마트

| 항목 | 상세 |
|------|------|
| **공식 파트너 API** | ❌ 외부 개발자용 공개 API 없음 |
| **연동 현황** | 배달대행 업체 대상 주문 연동만 존재 (보안 심사 필요) |
| **기술 구조** | MSA 기반 내부 API, 외부 공개 문서 없음 |
| **MVP 우회 전략** | B마트 딥링크 또는 검색 연결 |
| **실현 가능성** | ⭐⭐ — 초기에는 연동 불가, 사업 규모 확대 후 제휴 협상 필요 |

### 3.4 커머스 연동 종합 전략

```
Phase 1 (MVP):    쿠팡/컬리/B마트 검색 URL 자동 생성 → 사용자가 직접 구매
Phase 2 (성장기):  쿠팡 파트너스 API 연동 (어필리에이트 수익 확보)
Phase 3 (확장기):  마켓컬리·B마트 사업 제휴 → 직접 장바구니 연동
```

## 4. 레시피 파싱 기술 분석

### 4.1 한국 레시피 사이트 구조화 데이터 현황

| 사이트 | JSON-LD (Schema.org/Recipe) | HTML 구조 | 파싱 난이도 |
|--------|---------------------------|-----------|------------|
| 만개의레시피 | ❌ 미적용 | BeautifulSoup 크롤링 필요 | 중 |
| 해먹남녀 | ❌ 미적용 | HTML 파싱 필요 | 중 |
| 해외 레시피 사이트 | ✅ 대부분 적용 | JSON-LD 파싱 가능 | 하 |

**핵심 이슈:** 한국 사이트는 JSON-LD 구조화 데이터 채택률이 극히 낮음 (조사 대상 208개 국내 사이트 중 JSON-LD 사용 0건). 크롬 익스텐션에서 HTML 직접 파싱 또는 AI 기반 추출이 필요.

### 4.2 레시피 파싱 기술 스택 옵션

| 방식 | 장점 | 단점 | 추천도 |
|------|------|------|--------|
| **JSON-LD 파싱** | 표준화, 정확도 높음 | 한국 사이트 미지원 | 해외 사이트 전용 |
| **HTML 크롤링** | 모든 사이트 대응 | 사이트별 파서 개발 필요 | ⭐⭐⭐ |
| **AI 기반 추출** | 범용적, 구조 변경에 강함 | 비용 발생, 정확도 변동 | ⭐⭐⭐⭐ (장기) |
| **하이브리드** | JSON-LD 우선 → AI 폴백 | 구현 복잡도 증가 | ⭐⭐⭐⭐⭐ |

**추천:** 해외 사이트는 JSON-LD 파싱, 한국 사이트는 주요 5~10개 사이트용 HTML 파서 + AI 폴백의 하이브리드 전략.

## 6. 추천 플랫폼 전략

### 6.1 Phase별 로드맵

#### Phase 1: MVP 검증 (크롬 익스텐션 + PWA)

```
┌─────────────────────────────────────────────────────┐
│  크롬 익스텐션 (React + TypeScript + Vite)           │
│  ├── 웹 레시피 원클릭 저장                             │
│  ├── 재료 자동 추출 (HTML 파서 + AI 폴백)             │
│  └── Supabase에 레시피 데이터 저장                     │
│                                                       │
│  PWA 웹앱 (Next.js + Supabase)                       │
│  ├── 주간 식단 캘린더 (드래그 앤 드롭)                  │
│  ├── 장보기 리스트 (재료 자동 취합)                     │
│  ├── 쿠팡/컬리 검색 링크 자동 생성                      │
│  └── 반응형 모바일 최적화                               │
└─────────────────────────────────────────────────────┘
```

**이유:**
- 크롬 익스텐션은 "레시피 저장" 진입점으로 최적 (사용자가 레시피를 탐색하는 곳이 브라우저)
- PWA는 SEO 이점 + 앱 설치 없이 접근 가능 + 개발 비용 최소
- Supabase 무료 티어로 MVP 비용 최소화
- Next.js의 SSR/SSG로 레시피 콘텐츠 SEO 극대화

#### Phase 2: 성장기 (모바일 앱 확장)

```
┌─────────────────────────────────────────────────────┐
│  React Native 모바일 앱                               │
│  ├── PWA 기능 이식 + 네이티브 UX                       │
│  ├── 푸시 알림 (식사 리마인더)                          │
│  ├── 오프라인 장보기 리스트                              │
│  └── 쿠팡 파트너스 API 연동 (수익화)                    │
│                                                       │
│  기존 크롬 익스텐션 + PWA 유지                          │
└─────────────────────────────────────────────────────┘
```

**이유:**
- React Native는 JavaScript 인력풀이 넓고, PWA 코드의 로직 재사용 가능
- 앱스토어 존재감 확보로 사용자 획득 채널 확대
- 쿠팡 파트너스 매출 15만원 달성 후 API 전환

#### Phase 3: 확장기

```
┌─────────────────────────────────────────────────────┐
│  AI 기능 강화                                         │
│  ├── 냉장고 파먹기 모드 (pgvector 유사도 검색)          │
│  ├── 개인 맞춤 식단 추천                                │
│  └── 영양소 자동 분석                                   │
│                                                       │
│  커머스 통합                                           │
│  ├── 마켓컬리/B마트 사업 제휴                           │
│  └── 원클릭 장바구니 담기                               │
└─────────────────────────────────────────────────────┘
```

### 6.2 추천 기술 스택 총정리

| 레이어 | 기술 | 선정 이유 |
|--------|------|----------|
| **크롬 익스텐션** | React + TypeScript + Vite + Manifest V3 | 2025 표준 스택, 빠른 개발 |
| **프론트엔드 (PWA)** | Next.js 14+ (App Router) | SSR/SSG/ISR, SEO 최적화, React 생태계 |
| **스타일링** | Tailwind CSS | 빠른 UI 개발, 반응형 |
| **백엔드/DB** | Supabase (PostgreSQL) | 관계형 데이터 최적, 무료 티어, AI 벡터 |
| **인증** | Supabase Auth | 소셜 로그인 내장 |
| **파일 스토리지** | Supabase Storage | 레시피 이미지 저장 |
| **배포** | Vercel (Next.js) | Next.js 최적 호스팅, 무료 티어 |
| **모바일 (Phase 2)** | React Native | JS 코드 재활용, 넓은 인력풀 |
| **AI/검색** | pgvector + OpenAI API | 재료 기반 레시피 추천, 냉장고 파먹기 |

## 7. 리스크 및 대응 방안

| 리스크 | 영향도 | 대응 방안 |
|--------|-------|----------|
| 한국 레시피 사이트 크롤링 법적 이슈 | 높음 | robots.txt 준수, 사이트 제휴 추진, 사용자가 직접 저장하는 방식(크롬 익스텐션)으로 법적 리스크 최소화 |

| 마켓컬리/B마트 공식 API 부재 | 중 | 검색 링크 방식 + 사용자 수 확보 후 사업 제안 |

## 8. 결론

### 핵심 판단

1. **시장 타이밍이 유리함** — 한국 "직접 요리" 트렌드 강화 + 통합 플로우 서비스 부재
2. **MVP 구현 난이도가 낮음** — 크롬 익스텐션 + PWA 조합으로 소규모 팀이 충분히 구현 가능
3. **크롬 익스텐션 + PWA가 최적** — 사용자가 레시피를 찾는 곳(브라우저)에서 시작하여, 별도 앱 설치 없이 식단 관리까지 연결
4. **커머스 연동은 단계적 접근** — MVP에서는 검색 URL 방식으로 시작, 트래픽 확보 후 API/사업 제휴로 전환
5. **Supabase + Next.js가 기술 스택 최적해** — 관계형 데이터 모델, SEO, 비용 효율성, AI 확장성 모두 충족
